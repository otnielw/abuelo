Changelog

What's new in Algorithm Visualizer

v2.0 February 28, 2026
Landing Page + Graph Algorithms
Project Structure
  • Restructured the site from a single-page app into a multi-page architecture with a central landing hub
  • Renamed public/index.html (sorting page) to public/sorting.html to give it a dedicated URL
  • Created a new public/index.html as the landing page, now served at the root URL /
  • Created public/graphs.html for graph algorithm visualizations
  • No changes to server.js — Express static file serving handles the new structure automatically
Added: Landing Page
  • New visually engaging hub page at / that guides users to both algorithm sections
  • Animated hero section with gradient title "Algorithm Visualizer" and radial gradient pulse background
  • Two interactive feature cards in a responsive grid layout
  • Each card includes a self-running animated SVG preview — the sorting card shows a mini bubble sort, the graph card shows a mini BFS traversal
  • Algorithm tags (badges) on each card list all 5 algorithms available in that section
  • Hover effects with translateY lift and enhanced box shadow
  • Glassmorphism sticky navigation bar with links to both section pages
  • Fully responsive — cards stack vertically on mobile viewports
Added: Graph Algorithms Page
  • Brand new page with 5 interactive graph algorithm visualizations matching the sorting page's design language
  • SVG-based node-edge graph rendering with pre-computed node positions (no force layout needed)
  • Each algorithm has: animated visualization, color-coded legend, controls (Run/Reset + speed slider), live status text, complexity badges, and step-by-step explanation cards
  • Unique color gradient per algorithm for visual identity (accent bar on top of each card)

5 Graph Algorithms Visualized

BFS
Level-by-level traversal
DFS
Depth-first with backtracking
Dijkstra
Weighted shortest path
Prim's MST
Minimum spanning tree
A* Search
Heuristic grid pathfinding
  • BFS & DFS share the same 8-node unweighted graph for easy comparison between traversal strategies
  • Dijkstra & Prim's share the same 8-node weighted graph with edge weight labels displayed on each edge
  • Dijkstra shows live distance labels under each node (initially infinity, updated as edges are relaxed) and highlights the shortest path tree at the end
  • Prim's MST highlights candidate edges in yellow, the cheapest pick in pink, and the growing tree in green, with a running total weight counter
  • A* Search uses a completely different visualization: a 20x12 interactive SVG grid where users can click to place/remove walls, generate random mazes, and watch the open set (yellow), closed set (cyan), and final path (purple) unfold
Changed: Sorting Page
  • Moved from public/index.html to public/sorting.html
  • Added "Home" navigation link to return to the landing page
  • Added "Graph Algorithms" navigation link for cross-page navigation
  • All sorting algorithm visualizations and functionality remain unchanged
Technical Details
  • All pages use the same CSS design tokens (colors, fonts, spacing, card styles) for visual consistency
  • Graph algorithms use the same async/await + sleep + cancellation animation pattern as the sorting page
  • Graph page uses a shared drawGraph() function that handles both weighted and unweighted graph rendering with configurable node/edge colors and distance labels
  • A* grid uses SVG <rect> elements with mousedown/mousemove event handlers for wall drawing
  • All pages are fully self-contained HTML files with inline CSS and JavaScript — no external dependencies, no build step
  • Responsive across all screen sizes
